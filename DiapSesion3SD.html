📋 LEYES FUNDAMENTALES DEL ÁLGEBRA BOOLEANA

🔹 REGLAS 1-2: LEYES DE IDENTIDAD

1. A + 0 = A
2. A · 1 = A

El elemento neutro no afecta la variable

🔹 REGLAS 3-4: LEYES DEL ELEMENTO NULO

3. A + 1 = 1
4. A · 0 = 0

El elemento dominante absorbe cualquier valor

🔹 REGLAS 5-6: LEYES DE IDEMPOTENCIA

5. A + A = A
6. A · A = A

Una variable operada consigo misma no cambia

🔹 REGLAS 7-8: LEYES DEL COMPLEMENTO

7. A + Ā = 1
8. A · Ā = 0

Variable y su complemento son exhaustivos

🔹 REGLA 9: LEY DE INVOLUCIÓN

9. A̿ = A

Doble negación equivale a la variable original

⚡ LEYES AVANZADAS Y REGLAS ESPECIALES

🔄 LEYES CONMUTATIVAS

A + B = B + A
A · B = B · A

El orden de las variables no importa

🔗 LEYES ASOCIATIVAS

(A + B) + C = A + (B + C)
(A · B) · C = A · (B · C)

La agrupación de variables no afecta el resultado

📊 LEYES DISTRIBUTIVAS

A · (B + C) = A·B + A·C
A + (B · C) = (A+B)·(A+C)

Factorización y expansión de expresiones

🌪️ LEYES DE ABSORCIÓN

A + (A · B) = A
A · (A + B) = A

Eliminación de términos redundantes

🔥 TEOREMAS DE DE MORGAN

Los teoremas más importantes para simplificación de circuitos:

📜 PRIMER TEOREMA

(A + B)̄ = Ā · B̄

"La negación de una suma es igual al producto de las negaciones"

Equivalencia: NAND ↔ Negativa-OR

📜 SEGUNDO TEOREMA

(A · B)̄ = Ā + B̄

"La negación de un producto es igual a la suma de las negaciones"

Equivalencia: NOR ↔ Negativa-AND

🎯 Para más de dos variables:

(A + B + C + ...)̄ = Ā · B̄ · C̄ · ...
(A · B · C · ...)̄ = Ā + B̄ + C̄ + ...

🎯 Regla Nemotécnica:

"Rompe la barra, cambia el signo" 🔄

🛠️ 2.4.1 SIMPLIFICACIÓN MEDIANTE ÁLGEBRA BOOLEANA

📝 Ejemplo paso a paso: BD + B(D + E) + D̄(D + F)

Expresión original: BD + B(D + E) + D̄(D + F)
Expandir términos: BD + BD + BE + D̄D + D̄F
Aplicar A + A = A: BD + BE + D̄D + D̄F
Aplicar A · Ā = 0: BD + BE + 0 + D̄F
Resultado final: BD + BE + D̄F = B(D + E) + D̄F

🔧 FUNCIÓN ORIGINAL

BD + B(D + E) + D̄(D + F)

Costo: 7 compuertas lógicas

⚡ FUNCIÓN SIMPLIFICADA

B(D + E) + D̄F

Costo: 4 compuertas lógicas

🎯 Ahorro: 43% menos compuertas

🗺️ 2.5 MINIMIZACIÓN MEDIANTE MAPAS DE KARNAUGH

Método gráfico desarrollado por Maurice Karnaugh (1953) para simplificar expresiones booleanas

📋 VENTAJAS DEL MÉTODO

  • Visualización clara de patrones
  • Simplificación sistemática
  • Detecta grupos fácilmente
  • Minimiza errores humanos
  • Resultado óptimo garantizado

🎯 PRINCIPIO FUNDAMENTAL

Código Gray: Solo cambia 1 bit entre celdas adyacentes
  • Celdas adyacentes difieren en 1 variable
  • Grupos de 1s = términos simplificados
  • Tamaños válidos: 1, 2, 4, 8, 16...

📝 EJERCICIOS DE SIMPLIFICACIÓN

🎯 Ejercicio 1: Simplificar ABC + AB + AC

Paso 1: Factorizar A: A(BC + B + C)
Paso 2: Aplicar absorción B + BC = B: A(B + C)
Resultado: ABC + AB + AC = A(B + C)

📊 Ejercicio 2: (B + BC)(B + B̄C)(B + D)

Aplicar leyes de absorción y simplificación

Resultado: B(B + C + D) = B

Siempre verificar que la expresión original y simplificada den los mismos resultados

🔧 LAB4: PWM CON ARDUINO - INTRODUCCIÓN

🎯 ¿Qué es Arduino?

  • Microcontrolador: Pequeña computadora programable
  • Entradas digitales: Leen 0V (LOW) o 5V (HIGH)
  • Salidas digitales: Envían 0V o 5V
  • PWM: Simula señales analógicas

⚡ ¿Qué es PWM?

Pulse Width Modulation (Modulación por Ancho de Pulso)

Cambia el tiempo que la señal está en HIGH vs LOW
  • 20% HIGH: ≈ 1V efectivo
  • 50% HIGH: ≈ 2.5V efectivo
  • 80% HIGH: ≈ 4V efectivo

🔌 Pines PWM en Arduino UNO

Pines PWM: 3, 5, 6, 9, 10, 11 (marcados con ~)

💻 LAB4: ESTRUCTURA DEL CÓDIGO ARDUINO

📋 Partes de un programa Arduino

  • Variables globales: Se declaran al inicio
  • setup(): Se ejecuta UNA vez al inicio
  • loop(): Se ejecuta INFINITAMENTE

🎯 Funciones importantes

  • pinMode(): Configura pin como entrada/salida
  • analogWrite(): Envía valor PWM (0-255)
  • delay(): Pausa en milisegundos
// DECLARACIÓN DE VARIABLES (al inicio del programa) int BRILLO; // Variable para controlar el brillo (0-255) int LED = 3; // Pin donde está conectado el LED // FUNCIÓN SETUP - Se ejecuta UNA sola vez void setup() { pinMode(LED, OUTPUT); // Configura el pin LED como salida } // FUNCIÓN LOOP - Se ejecuta INFINITAMENTE void loop() { // Aquí va el código que se repite }

💡 Explicación: El programa Arduino tiene una estructura fija que SIEMPRE debe respetarse

🔍 LAB4: CÓDIGO COMPLETO EXPLICADO

int BRILLO; // Variable para el brillo (número entre 0 y 255) int LED = 3; // El LED está conectado al pin 3 (PWM) void setup() { pinMode(LED, OUTPUT); // El pin 3 envía señales (OUTPUT) } void loop() { // PRIMER BUCLE: Aumentar brillo de 0 a 255 for (BRILLO = 0; BRILLO < 256; BRILLO++) { analogWrite(LED, BRILLO); // Enviar valor PWM al LED delay(15); // Esperar 15 milisegundos } // SEGUNDO BUCLE: Disminuir brillo de 255 a 0 for (BRILLO = 255; BRILLO >= 0; BRILLO--) { analogWrite(LED, BRILLO); // Enviar valor PWM al LED delay(15); // Esperar 15 milisegundos } // El loop() termina y vuelve a empezar automáticamente }

🔄 Bucle FOR explicado

for (inicio; condición; incremento)

  • inicio: BRILLO = 0
  • condición: BRILLO < 256
  • incremento: BRILLO++

⚡ analogWrite() explicado

analogWrite(pin, valor)

  • 0: LED completamente apagado
  • 127: LED al 50% de brillo
  • 255: LED completamente encendido

🔌 LAB4: CONEXIÓN DEL CIRCUITO

📋 Componentes necesarios:

  • 1 Arduino UNO
  • 1 LED (cualquier color)
  • 1 Resistencia de 330Ω
  • 2 Cables jumper
  • 1 Protoboard

🔧 Pasos de conexión:

1. Conectar cable del pin 3 al protoboard
2. Conectar resistencia en serie con el LED
3. Conectar el otro extremo a GND (tierra)
4. Verificar polaridad del LED

⚠️ Importante:

LED tiene polaridad:

  • Ánodo (+): Pata larga → Resistencia
  • Cátodo (-): Pata corta → GND

La resistencia protege el LED

💾 LAB4: PASOS PARA PROGRAMAR ARDUINO

🚀 Proceso completo:

Paso 1: Abrir Arduino IDE en la computadora
Paso 2: Conectar Arduino UNO con cable USB
Paso 3: Verificar que encienda LED verde "ON"
Paso 4: Borrar código existente y escribir el nuevo
Paso 5: Verificar código (botón ✓) - guardarlo como "PWMconLed"
Paso 6: Subir a Arduino (botón →)
Paso 7: ¡Observar el LED parpadeando gradualmente!

✅ Resultado esperado:

  • LED aumenta brillo gradualmente
  • LED disminuye brillo gradualmente
  • Ciclo se repite infinitamente
  • Efecto "respiración" del LED

🛠️ Troubleshooting:

  • LED no enciende: Verificar polaridad
  • Error al subir: Revisar puerto COM
  • LED muy tenue: Cambiar resistencia
  • No compila: Revisar sintaxis
