📝 DISEÑO VHDL DEL SUMADOR

🔬 Arquitectura del Sumador 4 Bits

Implementación usando full adders interconectados en cascada para propagar el acarreo

🏗️ Sumador Binario de 4 Bits - Archivo Principal

Archivo: sumador_binario_poracarreo4bits.vhd

VHDL - Diseño Principal
library ieee;
use ieee.std_logic_1164.all;

entity sumador_binario_poracarreo4bits is
    port ( 
        a    : in  std_logic_vector(3 downto 0);
        b    : in  std_logic_vector(3 downto 0);
        cin  : in  std_logic;
        s    : out std_logic_vector(3 downto 0);
        cout : out std_logic
    );
end sumador_binario_poracarreo4bits;

architecture rtl of sumador_binario_poracarreo4bits is
    signal c0, c1, c2 : std_logic;
begin
    -- Bit 0
    s(0) <= a(0) xor b(0) xor cin;
    c0 <= (a(0) and b(0)) or (cin and (a(0) xor b(0)));
    -- Bit 1  
    s(1) <= a(1) xor b(1) xor c0;
    c1 <= (a(1) and b(1)) or (c0 and (a(1) xor b(1)));
    -- Bit 2
    s(2) <= a(2) xor b(2) xor c1;
    c2 <= (a(2) and b(2)) or (c1 and (a(2) xor b(2)));
    -- Bit 3
    s(3) <= a(3) xor b(3) xor c2;
    cout <= (a(3) and b(3)) or (c2 and (a(3) xor b(3)));
end rtl;

📊 Casos de Prueba del Testbench

Caso 1: a=0101, b=0011, cin=0 → s=1000, cout=0 (5+3=8)

Caso 2: a=1100, b=0111, cin=0 → s=0011, cout=1 (12+7=19)

Caso 3: a=1111, b=0001, cin=0 → s=0000, cout=1 (15+1=16)

Caso 4: a=1000, b=1000, cin=0 → s=0000, cout=1 (8+8=16)

Caso 5: a=1001, b=0110, cin=1 → s=0000, cout=1 (9+6+1=16)

Caso 6: a=0000, b=0000, cin=0 → s=0000, cout=0 (0+0=0)

Caso 7: a=0111, b=1000, cin=0 → s=1111, cout=0 (7+8=15)

Caso 8: a=0001, b=0001, cin=0 → s=0010, cout=0 (1+1=2)

🏗️ Arquitectura RTL vs Estructural

Este diseño usa arquitectura RTL directa en lugar de componentes separados

✅ Ventajas del Diseño RTL

  • Más compacto: Todo en un solo archivo
  • Mejor rendimiento: Sin overhead de componentes
  • Más legible: Flujo secuencial claro
  • Fácil síntesis: Herramientas optimizan mejor

⚡ Señales Internas de Acarreo

  • c0: Acarreo del bit 0 al bit 1
  • c1: Acarreo del bit 1 al bit 2
  • c2: Acarreo del bit 2 al bit 3
  • cout: Acarreo final de salida

🔧 Ecuaciones Implementadas

Para cada bit i:

  • Suma: s(i) = a(i) ⊕ b(i) ⊕ carry_in
  • Acarreo: carry_out = (a(i) ∧ b(i)) ∨ (carry_in ∧ (a(i) ⊕ b(i)))

Testbench que verifica el funcionamiento con múltiples casos de prueba

VHDL - Testbench
-- ================================================
-- TESTBENCH PARA SUMADOR BINARIO 4 BITS
-- Casos de prueba completos para verificación
-- ================================================

library ieee;
use ieee.std_logic_1164.all;

entity tb_sumador_binario_poracarreo4bits is
end tb_sumador_binario_poracarreo4bits;

architecture test of tb_sumador_binario_poracarreo4bits is
    -- Declaración del componente a probar
    component sumador_binario_poracarreo4bits
        port ( 
            a    : in  std_logic_vector(3 downto 0);
            b    : in  std_logic_vector(3 downto 0);
            cin  : in  std_logic;
            s    : out std_logic_vector(3 downto 0);
            cout : out std_logic
        );
    end component;
    
    -- Señales para conectar al DUT (Device Under Test)
    signal a_tb    : std_logic_vector(3 downto 0);
    signal b_tb    : std_logic_vector(3 downto 0);
    signal cin_tb  : std_logic;
    signal s_tb    : std_logic_vector(3 downto 0);
    signal cout_tb : std_logic;
    
begin
    -- Instanciación del DUT (Design Under Test)
    dut: sumador_binario_poracarreo4bits port map (
        a    => a_tb,
        b    => b_tb,
        cin  => cin_tb,
        s    => s_tb,
        cout => cout_tb
    );
    
    -- Proceso de estimulación con casos de prueba
    process
    begin
        -- Inicialización
        a_tb <= (others => '0');
        b_tb <= (others => '0');
        cin_tb <= '0';
        wait for 1 ns;
        
        -- CASOS DE PRUEBA SISTEMÁTICOS
        
        -- Caso 1: 5 + 3 = 8 (sin desbordamiento)
        a_tb <= "0101"; b_tb <= "0011"; cin_tb <= '0'; wait for 10 ns;
        
        -- Caso 2: 12 + 7 = 19 (con desbordamiento: 0011 + carry)
        a_tb <= "1100"; b_tb <= "0111"; cin_tb <= '0'; wait for 10 ns;
        
        -- Caso 3: 15 + 1 = 16 (desbordamiento máximo: 0000 + carry)
        a_tb <= "1111"; b_tb <= "0001"; cin_tb <= '0'; wait for 10 ns;
        
        -- Caso 4: 8 + 8 = 16 (desbordamiento: 0000 + carry)
        a_tb <= "1000"; b_tb <= "1000"; cin_tb <= '0'; wait for 10 ns;
        
        -- Caso 5: 9 + 6 + 1 = 16 (con acarreo de entrada)
        a_tb <= "1001"; b_tb <= "0110"; cin_tb <= '1'; wait for 10 ns;
        
        -- Caso 6: 0 + 0 = 0 (caso trivial)
        a_tb <= "0000"; b_tb <= "0000"; cin_tb <= '0'; wait for 10 ns;
        
        -- Caso 7: 7 + 8 = 15 (suma máxima sin desbordamiento)
        a_tb <= "0111"; b_tb <= "1000"; cin_tb <= '0'; wait for 10 ns;
        
        -- Caso 8: 1 + 1 = 2 (suma mínima)
        a_tb <= "0001"; b_tb <= "0001"; cin_tb <= '0'; wait for 10 ns;
        
        -- Terminar simulación
        wait;
    end process;
    
end test;
